Repast - Error Creating Style

Sat 29 March 2014

I recently began using Repast to code an agent-based model / simulation. The first thing I wanted to do when running the model was to edit the style of each agent so that I could distinguish between the two types of agents I had (the default in Repast is a blue square).

This message was not very helpful to me in figuring out what the problem was. After a long search I came across this post. It seems that Repast throws this message if you try and edit the agent style of a class of agents that does not have a getter method for a class variable.

Thus (for example) adding

public int dummyVariable= 0;

public int getDummyVariable() {
    return this.dummyVariable;
}

to the agent class definition solves the issue, and allows the agent style to be edited.
